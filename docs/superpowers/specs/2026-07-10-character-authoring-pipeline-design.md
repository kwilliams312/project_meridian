## 1. Overview

Contract в‘  defined the data vocabulary for characters and modular gear but left
two things deliberately open: the canonical **bone list** (`skeleton.defs.yaml`
`bones:` is an empty stub) and the **authoring toolchain** that produces
conforming assets. This spec fills both:

1. A **from-scratch, deterministic rig generator** producing the exact Godot
   `SkeletonProfileHumanoid` bone set + Meridian socket bones.
2. **Geoset mesh conventions** binding body meshes to contract в‘ 's 8 hide regions.
3. **Export-addon and import-validator extensions** that enforce rig/geoset
   conformance at both ends of the Blenderв†’glTFв†’Godot pipeline.
4. A **greybox blockout body** proving the chain end to end.
5. A **Meshy.ai intake tool** that streamlines AI-generated assets into
   policy-compliant, provenance-complete form вЂ” including **conversion of
   Meshy auto-rigs to the canonical rig**.

Owner decisions (brainstormed 2026-07-10): Approach A вЂ” Godot-profile-aligned
bone names, generated by a from-scratch script (no Rigify, no hand-maintained
binary); scope includes the blockout body but **no animations**; Meshy intake
includes rig conversion in v1.

### Goals

- Fill `skeleton.defs.yaml` `bones:` from a single authoritative source (the
  generator's bone table) with a drift-guard test.
- Free retargeting: profile-named skeletons auto-map in Godot's `BoneMap` /
  retarget stack (Art PRD В§4: "in-engine polish via Godot's AnimationTree/
  retarget stack").
- Every skeletal asset вЂ” hand-made or AI-intake вЂ” passes the same conformance
  gates (export-side and import-side).
- AI-generated assets flow from prompt to policy-compliant sidecar in one
  command, quarantined by the existing `restyle_status: pending` lint until
  restyled (Art PRD В§3.4 "kitbash to style").

### Non-goals

- Animations (idle/walk/run) вЂ” separate follow-on package.
- Face bones and the 1вЂ“2 optional morphs (A-03) вЂ” additive later; the bone
  budget (в‰¤120, art-prd В§2.1) reserves headroom.
- Appearance catalog content and real (non-blockout) bodies/gear вЂ” spec в‘¤.
- The client assembler вЂ” spec в‘Ў.
- Meshy retexture endpoints and non-humanoid (creature) rig conversion вЂ” future.
- Porting L080вЂ“L083 lints into mcc (tracked from the в‘  final review as a в‘Ј-adjacent
  note; this spec's validators are Python/addon-side and do not change mcc).

## 2. Reference rig вЂ” contract & generator

### 2.1 Bone contract

- **Core bones:** the 56 bones of Godot's `SkeletonProfileHumanoid`, exact
  names and hierarchy (Root/Hips/Spine/Chest/UpperChest/Neck/Head, arms with
  full fingers, legs with toes). The generator embeds this table as code; it is
  the single source of truth.
- **Socket bones:** one per contract-в‘  attach socket, named `socket_<socket>`:

  | Socket bone | Parent |
  |---|---|
  | `socket_main_hand` | RightHand |
  | `socket_off_hand`, `socket_shield` | LeftHand |
  | `socket_back`, `socket_ranged` | Chest |
  | `socket_hip_l`, `socket_hip_r` | Hips (sheathed weapons must not swing with leg bones) |

  63 bones total; в‰¤120 budget honored with headroom for face bones later.
- **Rest pose:** T-pose matching the Godot profile reference, so `BoneMap`
  auto-detection is trivial. 1 Blender unit = 1 m; -Z forward, Y up on export
  (art-prd В§4.2).
- **Proportions are parameters:** the generator takes a proportion profile
  (default: ardent male, ~1.8 m). Future races/sexes = same bone names, new
  proportion profile вЂ” the shared-skeleton contract from в‘ .
- `schema/content/skeleton.defs.yaml` `bones:` is populated with these 63
  names. Additive-only evolution (append face bones later; never rename).

### 2.2 Generator

- New package `tools/blender/meridian_rig/`:
  - `bones.py` вЂ” pure-Python bone table (names, hierarchy, rest transforms per
    proportion profile). Importable without Blender; unit-testable; the drift-
    guard compares it to `skeleton.defs.yaml`.
  - `generate_rig.py` вЂ” runs under `blender --background --python`; builds the
    armature from `bones.py`, exports `.glb` via the pinned export preset.
- **Reproducibility:** structural determinism, not byte determinism вЂ” Blender
  export embeds environment-dependent bytes. The conformance test asserts bone
  names/hierarchy/rest transforms via pygltflib on the committed `.glb`. The
  spec pins the Blender version for regeneration (documented in
  `tools/blender/README.md`); regenerating with the pinned version must produce
  a structurally identical file.
- **Outputs (committed, LFS):** `.glb` + IF-8 sidecar
  `art.char.ardent.male.skeleton` (class `character_model`,
  `source_tier: original` вЂ” the provenance is the script). The `.blend` is a
  throwaway build artifact, never committed, never hand-edited.

## 3. Geoset mesh conventions

- Body meshes are named **`geo_<region>_lod<N>`**, one mesh per region Г— LOD;
  `<region>` в€€ the 8 `skeleton.defs.yaml` geoset regions.
- LOD0 must cover all 8 regions (a body with a missing region is invalid);
  unknown region names are errors.
- Skinning: в‰¤4 influences per vertex, normalized weights.
- Gear meshes (armor) are NOT geoset-named вЂ” they are whole skinned pieces that
  *hide* regions via item@2 `worn.hides`; only bodies carry geoset names.

## 4. Export-addon extensions (`tools/blender/meridian_export`)

New export-time checks for skeletal classes (`character_model`, `armor_model`):

- E-rules (new band, mirroring the existing addon check style):
  - bone names вЉ† `skeleton.defs.yaml` `bones:` (the addon reads the YAML вЂ” no
    second copy of the list);
  - all 7 socket bones present when exporting a skeleton asset;
  - geoset naming/coverage on `character_model` body meshes (per В§3);
  - в‰¤4 influences/vertex, normalized weights;
  - transforms applied, unit scale, correct axes.
- Failures block export with actionable messages (bone name diff lists).

## 5. Import-validator extensions (`tools/validate_imports.py`)

New I-band rules (I020+), parsing committed `.glb`s with pygltflib (already
used by the budget validator):

- I020 вЂ” a `character_model` skeleton asset carries exactly the canonical bone
  set (names + hierarchy);
- I021 вЂ” skinned gear (`armor_model`) binds only to canonical bones;
- I022 вЂ” geoset naming/LOD0 coverage on body meshes;
- I023 вЂ” LOD chain presence per art-prd В§2.1 for skeletal assets
  (`import_hints.lod_policy: single` exempts blockouts/props explicitly).
- Empty `bones:` in skeleton.defs previously meant "no check" (contract в‘ );
  populating it turns these on вЂ” additive, no grandfathering needed because the
  only existing skeletal assets are NPC placeholder models whose sidecars are
  class `creature_model` (not covered by I020вЂ“I022).
- Runs inside content CI automatically (`validate_content.py` invokes this
  module in-process вЂ” no workflow edit).

## 6. Blockout body

- Script-generated greybox (`tools/blender/meridian_rig/generate_blockout.py`):
  primitive shapes per region, cut into the 8 geoset meshes
  (`geo_head_lod0` вЂ¦ `geo_feet_lod0`), simple weights to the canonical rig,
  low-hundreds of tris, `import_hints.lod_policy: single`.
- Ships as `art.char.ardent.male.base` + sidecar (class `character_model`,
  `source_tier: original`).
- Purpose: proves generator в†’ export checks в†’ `.glb` + sidecar в†’
  import validator в†’ mcc в†’ pck end to end. **No appearance catalog file** вЂ”
  catalog content is spec в‘¤'s boundary.

## 7. Meshy.ai intake (`tools/meshy/`)

A Python CLI streamlining AI asset generation into policy-compliant form.

### 7.1 Generate + fetch

- `meshy generate --text "..."` / `--image ref.png` в†’ Meshy text-to-3D /
  image-to-3D (previewв†’refine), task polling, `.glb` download.
- `MESHY_API_KEY` env only; never committed. **CI never calls the API** вЂ” all
  tests mock the client (testing rules: mock every external API).
- TD-09 precondition: Meshy's commercial-terms check is verified at time of use
  and recorded (tool+version per asset); the CLI refuses to run without an
  operator-confirmed `--terms-verified` flag or config equivalent.

### 7.2 Normalize + land + provenance

- Axis/scale sanity (Meshy emits Y-up glTF), `sm_`/`sk_` naming, placement
  under `content/<ns>/assets/**`.
- Auto-written IF-8 sidecar: `source_tier: ai`, `ai.tool: "meshy@<model-ver>"`,
  auto-generated `prompts_file` (exact prompt, params, task id вЂ” PRD В§3.2
  prompt hygiene), `origin_url` = Meshy task URL, **`restyle_status: pending`**.
  The existing lint blocks ai-tier `pending` assets from merging вЂ” intake
  output is structurally quarantined until restyle marks it `done`. No new
  policy invented; the В§3.4 pipeline enforced by construction.
- Budget pre-check via the pygltflib validators at intake вЂ” over-budget
  generations fail immediately, not at review.

### 7.3 Rig conversion (owner adjustment, 2026-07-10: in scope for v1)

Converts a Meshy auto-rigged humanoid to the canonical rig:

- `meshy convert-rig <asset>` runs a headless-Blender pass
  (`tools/meshy/convert_rig.py` under `blender --background`):
  1. Import the Meshy rigged `.glb`.
  2. Map Meshy bones в†’ canonical bones via a **versioned mapping table**
     (`tools/meshy/bone_map.yaml`, keyed by Meshy model version вЂ” their naming
     can drift between releases).
  3. **Re-pose** the Meshy rest pose to the canonical T-pose (per-mapped-bone
     rotation deltas, applied before binding).
  4. **Transfer weights** bone-by-bone onto the generator's armature
     (instantiated from `bones.py` вЂ” same source as the reference rig).
     Unmapped helper bones (twist/roll extras) merge their weights into the
     nearest mapped ancestor.
  5. Re-export; enforce в‰¤4 influences + normalization.
- **Unmapped bone names are a hard, listed error** вЂ” the table grows
  deliberately, never silently.
- **Objective gate:** converted output must pass the same В§4 export checks and
  В§5 import rules as hand-made assets (I021: binds only to canonical bones).
  Conversion has no private definition of "good enough".
- v1 scope: humanoid biped rigs only; creature rigs and Meshy retexture are
  future work.

## 9. Error handling / validation

- Generator: unknown proportion-profile key, non-pinned Blender version в†’
  hard error with the pinned version named.
- Export addon: bone-diff lists, missing sockets/geosets, weight violations вЂ”
  block export.
- Import validator: I020вЂ“I023 per В§5; `--imports error` posture in CI
  (existing behavior).
- Meshy CLI: API errors surface task id + status; timeout on polling;
  unmapped bones per В§7.3; missing `MESHY_API_KEY` / terms flag в†’ refusal
  before any network call.

## 10. Testing

- `bones.py` pure-python unit tests (hierarchy well-formed, 63 names, socket
  parents) + drift-guard vs `skeleton.defs.yaml` + raceName-style order guard.
- pygltflib structural tests on the committed rig/blockout `.glb`s
  (names/hierarchy/rest transforms вЂ” the "structural determinism" contract).
- One negative fixture per new E-/I-rule (repo convention).
- Meshy client fully mocked (httpx mock): generate/poll/download happy path,
  API-error paths, sidecar/prompts-file content assertions, terms-flag refusal.
- convert-rig: fixture Meshy-style `.glb` (checked-in, hand-made mini-rig) в†’
  conversion в†’ assert output passes I020/I021; unmapped-bone fixture в†’ hard
  error listing names.
- Blender-dependent steps (generator, convert-rig) run locally, not in CI;
  CI validates committed artifacts structurally (pygltflib) вЂ” same posture as
  the strudel render pipeline's local-render/CI-check split.
